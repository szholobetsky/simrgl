When a user reports "lost indentation" or similar whitespace-formatting bugs in files generated by `deepagent_task`/`alkonost` (or similar plain-text tree output), verify the raw bytes first (Python `repr()` on lines, check for CRLF vs bare LF) and check the depth-computation logic against the real tree structure before assuming a code bug exists.

**Why**: investigated a reported "we lost all indentation for submodules" bug in `deepagent_task.py`'s `tasks.md` output. Raw bytes had correct 2-space-per-level indentation, CRLF line endings were correct, and `depth = id.count(".") + 1` matched the real recursive tree structure exactly (0 mismatches across 27 real nodes, verified via a script walking `_child_ids`). The user was viewing the file in plain Windows Notepad, which visually failed to show the indentation; reopening in Notepad++ showed it correctly. No code bug existed at all — the file was correct throughout.

**How to apply**: before touching code in response to a formatting/whitespace bug report, do the byte-level + structural verification first (cheap, fast, decisive). If both check out, ask what tool/viewer the user is using before making any changes — plain Notepad is a known unreliable reference for these files. Related: [[project_alkonost_build]].
